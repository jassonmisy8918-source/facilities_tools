🏗️ 智慧排水综合管理平台 - 前端开发原型需求

1. 项目概述
*   项目名称：智慧排水综合管理平台
*   核心架构：1个总驾驶舱 + 6大业务子系统
*   交互模式：
    *   登录页：真实交互（表单验证、Token存储、错误提示）。
    *   主页及业务页：静态原型（左侧菜单导航，右侧iframe或静态页面展示，仅模拟UI，不调用后端接口）。

3. 页面交互需求 (UI/UX)

整体布局：
*   布局模式：Classic Layout (经典布局)。
*   左侧导航栏：宽度 200px - 240px，背景深蓝/黑色，支持多级折叠展开（至少支持 2 级菜单，部分系统如“设施一张图”有 3 级）。
*   顶部栏：包含系统 Logo (智慧排水)、用户头像/名称、退出登录按钮。
*   主内容区：Tab页签式（可选）或单页覆盖式，背景浅灰色。

具体交互逻辑：
1.  登录页 (/login)：
    *   表单字段：用户名（输入框）、密码（密码框）。
    *   功能：点击“登录”按钮。
    *   真实逻辑：前端需校验非空；模拟后端返回 Token（如 JWT），并将 Token 存入 localStorage 或 Vuex/Pinia。
    *   路由守卫：未登录用户访问 /home 或其他业务页时，强制跳转至 /login。
2.  主页 (/home)：
    *   用户登录成功后进入的第一个页面。
    *   默认展示：“运行驾驶舱” 的首页（大屏可视化风格，可放一张大屏背景图或简单的欢迎卡片）。
3.  业务页交互：
    *   点击左侧菜单，右侧加载对应的静态页面。
    *   静态要求：不需要真实数据，但需要画出表格、图表（ECharts 占位图）、地图（地图容器占位图）的样式，以体现“原型”感。
    *   面包屑导航：顶部显示当前路径（如：运行驾驶舱 > 专题图层 > 排水设施一张图）。

4. 技术栈建议 (供开发者参考)

*   框架：Vue 3 (推荐) 或 React 18。
*   状态管理：Pinia (Vue) 或 Redux Toolkit (React)。
*   UI 组件库：
    *   Vue: Element Plus 或 Ant Design Vue
    *   React: Ant Design 或 Material UI
*   路由：Vue Router / React Router (需配置路由守卫 beforeEach)。
*   可视化：ECharts (用于模拟图表展示)。
*   地图：由于是静态原型，地图部分可用 div + 背景图模拟，或引入简单的 Leaflet/OpenLayers 展示底图（无需叠加真实业务图层数据）。

5. 开发交付物清单

1.  Login.vue / Login.tsx：包含表单验证和模拟登录逻辑。
2.  Layout.vue / Layout.tsx：包含侧边栏、顶部栏、路由视图出口。
3.  Home.vue：驾驶舱首页。
4.  6个系统的目录结构及对应的静态页面组件（例如 AssetManagement.vue, Inspection.vue 等）。
5.  router/index.js：配置完整的路由表（包含 children 嵌套路由）和路由守卫逻辑。

💡 给开发者的特别提示
*   菜单层级深：文档中很多功能点有 3 级甚至 4 级标题（如 9.1.1.4.1.1.1. 放大）。在前端菜单配置中，建议将 9.1.1.4.1.1 (图形浏览) 作为二级菜单，将 放大、缩小 等作为三级菜单项，或者将三级以下的功能点合并展示在一个页面的 Tabs 中，避免菜单过深导致体验差。
*   驾驶舱优先：根据文档描述，驾驶舱是“指挥中枢”，建议优先开发驾驶舱的 UI，使用大屏数据展示风格（深色背景、科技感边框）。
*   静态数据：所有表格数据和图表数据可以使用 Faker.js 生成假数据，或者写死在代码中，目的是“看起来像在运行”。
